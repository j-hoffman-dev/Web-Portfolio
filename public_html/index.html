06.

in progress.

Oh boy, where to begin with this one?

In college I wrote a couple of game engines, but nothing this substantial. Typically small in scale, inefficient, and with little regard for maintainability. I wanted to build something bigger, faster, better in every way.

Having to contending with the shear scale of this program it soon became evident that I couldn't just write code without considering how it would (or wouldn't) interact with other parts of the engine. It demanded a new approach to how I program, it challenged me to change how I thought about code. Major bottlenecks of previous engines had been the use of libraries like Java AWT and poor design practices. So I decided to instead utilize low level libraries like OpenGL and subsequently overhaul engine architecture to accommodate by brushing up on my computing theory- the difference between the two is like night and day.

The decision to use Mario for tech demos just fit since I could indulge in creating art for one of my favorite franchises and keep project scope within reason. There's still a lot of work to do but I'm eager to push this new engine into production on a original IP as soon as a good idea pops up.
